## 1. 주말 갭 리스크와 24/7 자산 활용

### 1.1 문제 정의

```
미국 시장 타임라인 (한국 시간 기준):
─────────────────────────────────────────────────────────────────

월요일 06:00  ◄── 미국 금요일 마감
     │
     │  ███████████████████████████████████████████
     │  █           주말 공백 (약 63시간)          █
     │  █    이 기간에 무슨 일이 일어날지 모름     █
     │  ███████████████████████████████████████████
     │
화요일 23:30  ◄── 미국 월요일 개장 (갭 발생 가능)


주말 동안 발생 가능한 이벤트:
• 지정학적 긴장 (전쟁, 테러, 제재)
• 정책 발표 (G20, Fed 인사 발언)
• 기업 뉴스 (M&A, 실적 프리뷰)
• 자연재해
• 암호화폐 시장 급변동
```

│  • 알트코인: 극단적 투기 심리 측정                               │
│                                                                  │
│  Tier 2: 외환 시장 (일요일 저녁부터)                             │
│  ─────────────────────────────────                               │
│  • 일요일 18:00 (한국) 오세아니아 개장                           │

│  • AUD/USD: 위험 선호도, 중국 영향                               │

### 1.3 주말 갭 예측 모델

```
주말 BTC 움직임 → 월요일 미국 시장 갭 예측

가설:
• BTC 주말 -5% 이상 → 월요일 SPY 갭다운 가능성 ↑
• BTC 주말 +5% 이상 → 월요일 SPY 갭업 가능성 ↑
• BTC 횡보 → 특별한 신호 없음

검증 필요 사항:
┌─────────────────────────────────────────────────────────────────┐
│  상관관계 분석 (백테스팅)                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Period: 2020-01 ~ 현재                                          │
│                                                                  │
│  측정:                                                           │
│  • BTC 주말 수익률 (금 마감 → 일 마감)                           │
│  • SPY 월요일 갭 (금 종가 → 월 시가)                             │
│  • SPY 월요일 전체 수익률                                        │
│                                                                  │
│  조건부 분석:                                                    │
│  • BTC 급락 주말 vs 평온한 주말                                  │
│  • VIX 수준별 (고변동성 기간 vs 저변동성 기간)                   │
│  • 이벤트 유무 (FOMC 주간, 실적 시즌 등)                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.5 주말 알림 시스템

```
알림 트리거 조건:
─────────────────────────────────────────────────────────────────

Level 1 - INFO (참고용):
• BTC 주말 변동 ±3% 이상
• 주요 뉴스 발생 (키워드 탐지)

Level 2 - CAUTION (주의):
• BTC 주말 변동 ±5% 이상
• 복수의 24/7 자산이 같은 방향으로 움직임
• FX 갭이 평소보다 큰 경우

Level 3 - WARNING (경고):
• BTC 주말 변동 ±10% 이상
• 스테이블코인 디페깅 조짐
• 복합적 리스크 신호

Level 4 - ALERT (긴급):
• 시스템적 위험 신호
• 거래소 장애/해킹
• 지정학적 위기 발생

알림 채널:
• Slack/Telegram 봇
• Email (Level 3 이상)
• SMS (Level 4만)
```

### 2.2 프로파일별 파라미터

|----------|--------------|----------|------------|
| 최대 포지션 크기 | 50% | 80% | 100% |
| 단일 자산 최대 비중 | 10% | 20% | 40% |
| Stop-loss | -3% | -7% | -15% |
| Take-profit | +5% | +15% | +30% |
| 레버리지 | 없음 | 1.5x까지 | 3x까지 |

| 신호 임계값 (진입) | 80% | 65% | 55% |

### 4.3 신호 충돌 해결

```
여러 신호가 상충할 때:
─────────────────────────────────────────────────────────────────

Case 1: 같은 자산, 다른 시간 지평
┌─────────────────────────────────────────────────────────────────┐
│  단기 신호: 매도 (과매수, RSI 높음)                              │
│  장기 신호: 매수 (저평가, 상승 추세)                             │
│                                                                  │
│  해결: 시간 지평별 분리 운용                                     │
│  • 장기 포지션 유지                                              │
│  • 단기적으로 신규 매수 자제 또는 일부 익절                      │
└─────────────────────────────────────────────────────────────────┘

Case 2: 다른 자산 클래스 신호 충돌
┌─────────────────────────────────────────────────────────────────┐
│  주식 신호: Risk-on (매수)                                       │
│  채권 신호: Risk-off (안전자산 선호)                             │
│                                                                  │
│  해결: 선행지표 우선순위 적용                                    │
│  • 신용 시장(채권) 신호가 주식보다 선행                          │
│  • 채권 신호 우선, 주식 포지션 축소 고려                         │
│  • 또는 분산: 주식 + 채권 동시 보유                              │
└─────────────────────────────────────────────────────────────────┘

Case 3: 모델 vs 내재가치 충돌
┌─────────────────────────────────────────────────────────────────┐
│  모델 신호: 매수 (통계적 패턴)                                   │
│  내재가치: 고평가 (버블 가능성)                                  │
│                                                                  │
│  해결: 보수적 접근                                               │
│  • 포지션 크기 축소                                              │
│  • 좁은 Stop-loss                                                │
│  • 또는 진입 보류                                                │
└─────────────────────────────────────────────────────────────────┘

신호 충돌 시 기본 원칙:
• 의심스러우면 하지 않는다 (No action is an action)
• 리스크 관리 신호가 수익 신호보다 우선
• 장기 신호가 단기 신호보다 우선 (트레이딩 아닌 경우)
```
